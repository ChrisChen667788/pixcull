错误上报政策

默认关闭。开启后,只在你点 “立即提交一次错误报告” 或者(将来)程序崩溃时才会把数据外发。每次发送什么、发到哪里都对你透明。

会上报的内容

  • App 版本号
  • 操作系统类别 / 版本(如 Darwin · arm64 · 24.x)
  • Python 版本
  • 最近一次 stderr 日志的末尾 200 行(脱敏后)

脱敏规则(发送前自动应用)

  • /Users/<name> → /Users/<redacted>
  • /home/<name> → /home/<redacted>
  • C:\Users\<name> → C:\Users\<redacted>
  • sk-... DeepSeek / OpenAI API key → sk-***
  • Bearer <token> → Bearer ***
  • hf_... Hugging Face token → hf_***
  • AKIA... AWS access key → AKIA***
  • 邮箱地址 → <email>

绝不会上报

  • 你的图片或图片路径
  • 图片字节数据
  • License token
  • 你写的 rubric 标注 / 评注文字
  • /annotation 或 /export 的请求体

上报地址

你在管理面板里自己填一个 endpoint URL。如果留空,即便 “开启” 也只是 dry-run(本地拼好 payload 但不发送),你能在浏览器里看到完整内容。

官方目前没有共享 endpoint — 这是骨架,等真正的 Sentry / PostHog 搭建好了才会启用。

关闭后

改回开关或者把 endpoint 留空都立即生效。设置存在 ~/Library/Application Support/PixCull/config.json 里, 你可以直接编辑或删除。
