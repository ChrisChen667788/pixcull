# ROADMAP v2.11 — 透明度的可发现性 + 解释深化 charter

> **触发 / 承接**:v2.9 + v2.10 造出了一整套「透明度」功能——可调相似度滑块、
> Scenes 时序分组(导航条 + 内联 header)、人脸 Close-ups(+点击定位)、判定
> glass box。但**做出来 ≠ 用户找得到**。在 v2.10 抓图时实测发现:近重复 /
> Scenes 的切换 pill 被挤进**横向溢出的工具栏**(`#filters` overflow-x),在右边
> 屏外——Playwright 直接报「element is not visible」。**功能强但入口藏起来了**,
> 这是典型的「工程师把能力做完就算了、没管用户怎么发现」。
>
> v2.11 = 让透明度工具**被看见、被理解**:修可发现性 + 深化解释。

## 北极星

延续 v2.8 反思的克制纪律 + v2.9 的透明主张,再加一条:**可发现性是功能的一部分**。
一个用户找不到的滑块,等于没做。

## 现状诊断(实测)

- **工具栏溢出**:`#filters` 是 `overflow-x` 单行,功能多了之后(决策/视图/近重复/
  Scenes/语义搜索/导出…)pill 被推到屏外,右侧的近重复折叠 + 🎬 时序场景**默认不可见**。
- **入口零提示**:相似度滑块只在折叠激活后出现;Scenes/Close-ups/glass-box 没有
  任何首次引导,老用户靠探索、新用户根本不知道存在。
- **glass-box「一句话理由」偏浅**:当前取 `rationale 首句 / 首条信号`,没有真正
  落到「哪一轴拖了后腿 / 哪一轴最强」的逐轴归因。

## P0 — 可发现性(最高杠杆)

### P0-1 · 工具栏不再藏功能(实测 bug)
- **What**:`#filters` 工具栏重排,让近重复 / Scenes / 语义搜索等透明度入口在常见
  视口(≥1280)**不溢出屏外**。方案:按职能分组 + 换行(flex-wrap)或把次要项收进
  「更多 ⋯」overflow 菜单(关键透明度入口留在一行可见区)。
- **Why**:v2.10 实测——这些 pill 在 1440px 下都在屏外,用户发现不了。
- **Verify**:Playwright 在 1280/1440/1680 三档视口断言 `.neardup-toggle` /
  `.scenes-toggle` 的 `boundingBox` 在视口内且 visible。

### P0-2 · 透明度三件套首次微提示
- **What**:首次进入结果页时,对「🎬 时序场景 / ≈ 近重复 / 🔍 判定理由」给一次性
  轻提示(coachmark / 脉冲点),`localStorage` 记忆,看过不再出现。复用既有
  `.onboard-tip` 机制 + `pixcull_seen_*` key 模式。
- **Why**:强功能 + 零提示 = 实际使用率接近 0。

## P1 — 解释深化

### P1-1 · glass-box「为什么」逐轴归因
- **What**:把判定 glass box 的一句话理由从「首句/首信号」升级为**逐轴驱动**——
  指出最强轴 + 最弱轴(如「构图 4.8 撑分,光线 2.5 拖后腿」),数据来自 `rubric_stars`。
  纯前端可算(已有逐轴星级),不依赖新后端。
- **Verify**:对 keep/maybe/cull 三类样本各看一句生成的理由是否点中真实最强/最弱轴。

### P1-2 · Close-ups 入口提示
- **What**:👤 close-ups 切换只在检测到人脸时出现(已实现);加一个极轻的
  「N 张脸」角标 / 首次提示,让用户知道可以看人脸特写。
