# PixCull v1.0 — Release Gate + Plan

> **Status:** all v0.4 → v0.13 charters complete.  v1.0 is a release
> gate, not a new-feature version: we pack ~70 slices into a single
> tagged release.

## Release gate criteria

The gate is a single multi-dimensional checklist.  Every line ships
green before `git tag v1.0.0` is allowed.

### 1. Distribution (3 platforms × auto-update)

- [ ] **macOS** — `brew install --cask pixcull` real and working
- [ ] **macOS** — Sparkle EdDSA-signed updates flowing through
      `appcast.xml`
- [ ] **Windows** — `.msi` signed via SignPath OSS, SmartScreen
      no longer flags "publisher unknown" (requires 30-day
      reputation aging)
- [ ] **Linux** — AppImage GPG-signed, AppImageUpdate delivering
      increments
- [ ] **GitHub release v1.0.0** carries all three signed
      artefacts + merged appcast.xml
- [ ] ≥ 100 users have successfully installed and run for at least
      one wedding-scale (1000+ photo) session

External-blocked:
- Apple Developer enrolment (USD 99/yr + 24-72h approval)
- SignPath OSS sponsorship (free; 1-2 week approval)
- GPG release key ceremony (offline, ~30 minutes)

`docs/DISTRIBUTION-CHECKLIST.md` is the runbook.

### 2. ML evaluation on real goldenset

- [ ] `make goldenset` builds goldenset/v0.11/ (or v0.<latest>/)
      with ≥ 2,000 rows across ≥ 5 verticals
- [ ] V3 rescorer trained on the real goldenset, NOT synthetic data
- [ ] `recall@5 ≥ baseline + 5%` on the goldenset (v0.11 target was
      +3%; v1.0 raises the bar)
- [ ] `ci_rescorer_regression.py` gates every PR against the v3
      baseline
- [ ] `docs/RESCORER-V3-RESULTS.md` regenerated by `eval_rescorer.py`
      with real numbers, NOT placeholders
- [ ] No vertical regresses by more than 2% recall@5
- [ ] Bias audit shows no > 2σ outliers in cull rate (currently 1.5σ)

### 3. Multi-user / real-time collaboration

- [ ] LAN multiplayer (v0.10) zero-config via mDNS — ship-as-is
- [ ] WebRTC datachannel (v0.11-P0-3) brings same-LAN presence to
      sub-100ms end-to-end
- [ ] WAN STUN/ICE NAT traversal ≥ 80% success rate (field-test
      across 10+ households)
- [ ] iOS Companion (v0.10 + v0.12 haptic) syncs streaming bursts
      with 0 frame loss on test rig
- [ ] Conflict-resolution UI (v0.10 + v0.13-P1-3) handles every
      tested same-photo, same-time scenario

### 4. Commercial layer

- [ ] License CLI mints valid signed JSON (v0.11-P0-4 — done)
- [ ] Stripe webhook + signed-license delivery (v0.12-P0-4 — done)
- [ ] 微信支付 webhook ready (v0.12-P0-4 — needs production RSA cert)
- [ ] Decision point: either
      - [ ] **Ship Studio plan paid** — ≥ 50 active subscribers
        within 30 days of v1.0 release, OR
      - [ ] **Ship free-only** — license CLI still in place for
        future activation, no paid plan promotion at v1.0
      Either is a valid v1.0 — the choice signals the
      commercialisation timeline.

### 5. AI judgment transparency

- [ ] Lightbox `A` key surfaces 6-axis attribution heatmap on every
      photo (v0.13-P0-1 — done)
- [ ] Inspector "🔭 视觉" chip pops style-ref distance breakdown
      (v0.13-P1-2 — done)
- [ ] Counterfactual chip ("+0.08 if rule-of-thirds") on every
      photo with a non-default composition (v0.13-P0-2 — done)
- [ ] Maybe-band cards (0.45..0.55) hover-explain why (v0.13-P0-3
      — done)
- [ ] `/admin/bias` returns no findings above 1.5σ for the v1.0
      release-candidate goldenset
- [ ] NL explainer functional on every supported platform (template
      fallback always; LLM optional)
- [ ] All explanations 100% local — no cloud dependency in any
      Inspector chip

### 6. Internationalisation

- [ ] ≥ 13 locales shipped (current: 13 — at gate)
- [ ] RTL layout verified for ar_SA (lightbox toolbar mirroring,
      scrubber direction, marquee handles)
- [ ] All locale files updated to the v1.0 string set (no missing
      translations; `key` returned-on-miss is a release blocker)

### 7. Brand + design polish

- [ ] DESIGN-AUDIT-2027Q3 score ≥ 4.4 / 5 — **currently 4.4 / 5**
- [ ] Brand identity (logo, gradient, font stack) used consistently
      across /upload, /history, /tether, /admin, /companion,
      /results, share pages, every export
- [ ] All hero-reveal animations covered by reduced-motion
      preference (currently true)

### 8. Dogfood (real-world production runs)

- [ ] Maintainer dogfooded ≥ 3 wedding-scale runs (each 1,500+
      photos) on v1.0-RC
- [ ] ≥ 3 external photographers ran ≥ 1 wedding each on v1.0-RC
- [ ] No P0 / data-loss bug filed in the final 4 weeks before tag
- [ ] All P0 / P1 issues from the gate filed to v1.0.x patch milestone

### 9. Documentation

- [ ] README.md updated with v1.0 install commands + signed
      GPG fingerprint
- [ ] `docs/ARCHITECTURE.md` reflects every v0.11-v0.13 change
- [ ] `docs/RELEASE-NOTES-v1.0.md` written: every notable
      slice since v0.10 release
- [ ] ModelScope / HuggingFace mirrors updated (v0.11 sync flow
      `make modelscope-sync`)

## Post-v1.0 commitments

After v1.0 ships, the project enters **maintenance mode** for at
least 1 calendar quarter:

- Quarterly minor releases (1.0.1, 1.0.2, ...) — security + compat
  fixes only
- No new charters until 2028 Q1
- ~3 hr/wk on bug triage + community support (Discord / GitHub
  issues / 即刻)
- Field-collected hard examples flow into goldenset/v1.0-aug/ via
  `make goldenset-aug-cron` (v0.13-P1-4)

## v2.0 horizon

Candidates documented in `docs/DESIGN-AUDIT-2027Q3.md`:

1. **Video culling / reel selection** — same rescorer stack on
   video frames; 8-12 weeks
2. **Generative compositing** — best-of composite from burst
   sequences (Apple's "Photo Stacks" but for photographers); 12-16 weeks
3. **Enterprise SSO + audit log** — SAML / OIDC + Studio plan
   billing dashboard; 4-6 weeks (requires a billing engineer)

v2.0 charter drafted no sooner than 2028 Q1 (per the post-v1.0
maintenance commitment above).
