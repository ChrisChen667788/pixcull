## Reading the numbers

The DSP heuristic detector (`scoring/audio_events.py`) is **weak on real
clips**: applause F1 = **0.00** (it fires no applause on 20 real
clapping clips), laughter F1 = 0.15, macro-F1 = **0.075**. That is the
honest motivation for v2.2-P0-1: a learned tagger should comfortably
clear this bar. The DSP path stays as the always-available fallback.

## How to reproduce

```bash
# 1. fetch a small labelled eval set (ESC-50 subset → folder-per-class)
#    laughter/ applause/ none/   (local only — never committed)
# 2. DSP baseline:
python scripts/eval_audio_tagger.py --eval-dir /tmp/pixcull_audio_eval
# 3. with a learned model (drop the verdict + ΔF1 columns in):
python scripts/eval_audio_tagger.py --eval-dir /tmp/pixcull_audio_eval \
    --model ~/.pixcull/models/audio_tagger.onnx --out docs/AUDIO-TAGGER-EVAL.md
```

A model installed at `~/.pixcull/models/audio_tagger.onnx`
(+ `.labels.json`) is picked up automatically by `scoring/audio_tagger.py`
and by `pixcull models path audio-tagger` (v2.2-P1-2).

## Learned model — status (pending a verified ONNX)

The eval harness + DSP baseline are done; the learned half needs a real
ONNX. As of this writing there is **no clean, directly-downloadable,
contract-compatible YAMNet/PANNs ONNX**: the public YAMNet artefacts are
TFLite / SavedModel, and YAMNet's I/O (waveform-in `[N]` → `[frames,521]`)
differs from the current `OnnxTagger` contract (`[N, frame]→[N, classes]`),
so wiring it also needs a small waveform-input adapter + an AudioSet→kind
label map. Options to finish: (a) convert YAMNet→ONNX locally with
`tf2onnx` (pulls the TensorFlow stack), or (b) drop a verified ONNX at the
path above. Either way it's then a one-command eval + promote.

## Eval set & attribution

ESC-50 (Piczak, *ACM MM* 2015), CC BY-NC 3.0 — a subset of the
`laughing` / `clapping` classes (+ negatives) is fetched **locally** for
evaluation only and is **not** redistributed in this repo (consistent
with the repo-hygiene "eval data is local-only" rule). ESC-50 has no
generic *music* class, so `music` is not covered by this set.
