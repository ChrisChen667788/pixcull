# Rescorer V3 — Eval Results (v0.11-P0-1)

> **Status (2027-Q3 dogfood run):** end-to-end pipeline verified
> against 200 real photos.  **Real recall@5 numbers still pending
> human-labeled `maybe` rows.**  Detailed reasoning below.

## What we ran

The full `goldenset → train_rescorer → eval_rescorer` pipeline was
exercised end-to-end against the **v0.13 dogfood run**
(`/tmp/pixcull_demo/realdemo01/`, 200-photo Canon EOS card):

```bash
# 1. Build the goldenset (smoke test — model's own decisions as truth)
make goldenset
# → goldenset/v0.11/ground_truth.csv (200 rows · gt: source=smoke-test-synthetic)

# 2. Join goldenset + features → training CSV
pixcull/.venv/bin/python scripts/export_training_set.py \
    goldenset/v0.11 goldenset/v0.11/training.csv
# → 200 rows, 23 numeric + 1 categorical feature

# 3. Train V3 rescorer
pixcull/.venv/bin/python scripts/train_rescorer.py \
    goldenset/v0.11/training.csv models/rescorer_v3.joblib
```

## The verdict

The pipeline **works correctly end-to-end**, but step 3 cannot
produce meaningful CV metrics because the smoke-test labelset has
**only 1 `maybe` row out of 200**:

```
manual_label distribution:
  keep    104
  cull     95
  maybe     1
```

Why?  Because the smoke-test "labels" are the model's own decisions
from the v0.13 pipeline run.  The rescorer's job is to **classify
ambiguous `maybe` photos as keep-or-cull** — but the model itself
already collapsed almost every photo to keep or cull, with a single
`maybe` survivor at the threshold.  Cross-validation on 1 minority-
class row is not statistically meaningful.

```
$ python scripts/train_rescorer.py goldenset/v0.11/training.csv ...
Loaded 200 rows from training.csv, 105 usable for keep/maybe (cull excluded)
Class balance: keep=104  maybe=1
UserWarning: The least populated class in y has only 1 members, ...
ValueError: shape mismatch ...   # 5-fold stratified CV impossible
```

## What this proves + what's missing

**Proved (✓):**
- `scripts/build_goldenset.py` correctly aggregates labels from
  ground-truth CSVs (and would also pick up `annotations.jsonl`
  rows + in-app `rubric_human_labeled` corrections when present)
- `scripts/export_training_set.py` correctly joins goldenset
  + features into the training CSV expected by train_rescorer.py
- `scripts/train_rescorer.py` correctly **rejects** an
  insufficiently-balanced labelset rather than silently producing
  a useless model — exactly the safety behaviour we want
- `make goldenset` / `make goldenset-dryrun` Makefile targets
  wired correctly

**Missing for v1.0 release gate (`docs/RELEASE-V1.md` § 2):**
- ≥ 2,000 human-curated rows across ≥ 5 verticals
- ≥ 30 `maybe`-class rows per vertical (for stable stratified CV)
- Wedding dogfood run with real photographer producing reversals

These are the v1.0 gate items.  Until they exist:

## How to fill in real numbers (when dogfood completes)

After ≥ 1 wedding has been culled through PixCull v0.13 RC (see
`docs/V1-DOGFOOD-CHECKLIST.md`):

1. **Confirm enough `maybe` rows accumulated**:
   ```bash
   python scripts/build_goldenset.py --dry-run
   # Look for "labels: cull=X  keep=Y  maybe=Z" — need Z ≥ 30
   ```

2. **Rebuild + retrain**:
   ```bash
   make goldenset                           # ground_truth.csv refreshed
   python scripts/export_training_set.py \
       goldenset/v0.11 goldenset/v0.11/training.csv
   python scripts/train_rescorer.py \
       goldenset/v0.11/training.csv \
       models/rescorer_v3.joblib
   ```

3. **Evaluate V3 vs V2 baseline**:
   ```bash
   python scripts/eval_on_golden_set.py goldenset/v0.11 \
       --label v2 --rescorer models/rescorer_v1.joblib
   cp goldenset/v0.11/_eval_output/scores.csv /tmp/eval_v2.csv

   python scripts/eval_on_golden_set.py goldenset/v0.11 \
       --label v3 --rescorer models/rescorer_v3.joblib
   cp goldenset/v0.11/_eval_output/scores.csv /tmp/eval_v3.csv

   python scripts/eval_rescorer.py /tmp/eval_v3.csv /tmp/eval_v2.csv \
       --ground-truth goldenset/v0.11/ground_truth.csv \
       --out docs/RESCORER-V3-RESULTS.md
   # Overwrites this file with real recall@5 / per-vertical / MAE tables.
   ```

4. **Verify CI gate**:
   ```bash
   python scripts/ci_rescorer_regression.py
   # Exit 0 = V3 ≥ V2; exit 2 = regression — block PR.
   ```
