# v2.25 — 功能:n 路 A/B 比较(自由配对累积成集)

> 迭代队列 #1(功能)。审计 Q3 UX 缺口:自由配对比较严格两两——点第三张就
> 重锚,一个 pro 要比 4-5 张跨连拍组的近似片得做多轮两两。

## 问题

compare modal(`openCompareCustom`)本已是 **n 格**(接受任意 filename 列表、
`cmpBody.dataset.n` 驱动 CSS),唯一的两两瓶颈是**入口** `pinForCompare`:
选到第 2 张就立即开图,没法累积第 3-5 张。

## 方案(v2.25-P0)

- `_compareA`(单个)→ **`_compareSet`(有序集)**:每次选片 **toggle** 进/出集,
  **不再自动开图**。
- **托盘**从"已选 A: fn"改为"**已选 N 张 · 比较 N 张 · 清空**",N≥2 时露出比较按钮。
- **开图**:点托盘「比较」按钮,或**回车**(有待比较集≥2 时;否则回车仍开标注),
  或 lightbox 里 `c` 键加入后≥2 即开(保留快速 1:1-核对-配对流)。**Esc/清空**空集。
- 所有入口(`.card-cmp-btn` / Shift+点击 / `c` 键 / 卡片比较按钮)统一走累积。
- 新增 8 个 i18n 键 × 13 locale(托盘计数/提示/按钮/清空 + 2 toast + modal
  标题/说明);全部带 `{n}` 占位;locale 切换重译托盘。顺手把 compare modal
  标题/说明也接进 `_t()`(之前硬编码中文,英文托盘选完弹中文标题不一致)。

## 验收(Playwright 实测)

- 点 3 个比较按钮 → 托盘 **"3 selected"**、按钮 **"Compare 3"**、3 卡高亮环。
- 再点第一个 → **"2 selected"**(toggle 移除生效)。
- 补回 + 点比较 → modal 开出 **3 格**(`data-n=3`、标题 "A/B 比较 (3 张)"),
  托盘清空。
- i18n:en-US 浏览器全程英文(托盘 "3 selected" / "Compare 3")。
- 门禁绿(5 预期 skip);n 路比较守卫 +3 测试;版本 2.24→2.25 lockstep。
