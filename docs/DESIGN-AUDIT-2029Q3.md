## 三、诚实偏差(扣分项,不掩饰)

1. **P1-1 Scenes 用导航条,不是内联分段 header。** 章程原写「网格内插时序分段
   header」。实际 grid render 有 小/中/超大批量 + IntersectionObserver 占位 三条
   路径,内联插 header 风险高。改为**导航条 + 单段筛选**(复用 semSearch 文件名集
   交集),交付了「时序分段 + header(时间范围/张数/keep)+ 叙事跳转」,但不是逐段
   分隔的连续叙事流。**留作 v2.10 增强:小/中批量走内联 header,超大批量退化导航条。**
2. **P0-1 人脸 Close-ups 缺浏览器实拍截图。** 本机 Playwright 不稳 + 真人脸截图
   触及第三方 PII(hygiene 红线),故后端端到端验证充分、前端仅构建/语法级验证。
   **gallery 截图见下节的取舍。**

## 四、跨界面一致性 + palette guard

- **palette guard 通过**:v2.9 新增 CSS 全部走 OKLCH token(`--accent / --base /
  --surface-* / --c-success` 等);仅有的 brass/espresso 字面量是**逐字镜像**既有
  zen-toggle 模式(`rgba(196,185,169,…)` 本就是 `--accent` 值,全仓 10+ 处在用),
  无 indigo/pink 回潮、无错配色漂移。
- **一致性**:滑块/场景 chip/glass box 三处都用编辑暖 token + 既有圆角/留白节奏;
  人脸轨复用 lightbox 焦点陷阱(不抢 Tab,沿用 v2.8.1 freeze 教训);Scenes 复用
  既有过滤链路(不动脆弱 render)。
